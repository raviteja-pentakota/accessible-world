Lesson 1: Introduction to HTML

Jump to video lesson

HTML stands for HyperText Markup Language. It is the standard language used to create and design web pages. HTML provides the basic structure of a web page by using elements like headings, paragraphs, links, images, lists, and more. These elements are represented by tags such as <h1>, <p>, <a>, <img>, and others.

Learning HTML is essential for anyone who wants to build websites or become a web developer. It is the foundation of all web content. Without HTML, there would be no way to display text, images, or multimedia on a webpage. Understanding HTML also helps in learning CSS and JavaScript, which are used to style and add interactivity to web pages.

HTML was first developed by Tim Berners-Lee in 1991 while he was working at CERN (European Organization for Nuclear Research). He created it as a way to share documents over the internet using hyperlinks. Since then, HTML has gone through many updates and improvements, with the latest version being HTML5.

Lesson2: Basic HTML code

Jump to video lesson

Every HTML web page starts with a special declaration: <!DOCTYPE html>. This tells the web browser that the document is written using HTML5, the latest version of HTML. It helps browsers understand and display the content correctly.

After the doctype, the structure begins with the <html> tag, which wraps the entire content of the page. Inside the HTML tag, there are two main sections: <head> and <body>.

The <head> section contains important information about the web page that is not visible to users, such as the page title, character encoding, and links to other resources. The <title> tag, which is placed inside the head section, sets the text shown in the browser’s title bar or tab.

The <body> section includes all the content that appears on the web page, like text, images, links, and buttons. Everything the user interacts with goes inside this tag.

HTML elements usually have an opening tag and a closing tag. The closing tag has a forward slash / before the tag name. For example, <title> is the opening tag and </title> is the closing tag. This structure tells the browser where each element starts and ends.

Lesson3: Headings and paragraphs

Jump to video lesson

Headings and paragraphs are the basic building blocks of content in an HTML page. They help structure the content in a readable and meaningful way. HTML provides six levels of headings, from <h1> to <h6>. The <h1> tag is used for the main heading of the page, and it is the largest in size. The <h6> is the smallest heading. You should use headings in order to show the importance or structure of content—just like titles, subtitles, and section headings.

Paragraphs in HTML are created using the <p> tag. This tag is used to add blocks of text. When you write text between the opening <p> and closing </p> tags, the browser automatically adds some space before and after the paragraph to separate it from other elements.

If you want to break a line within a paragraph (without starting a new paragraph), you can use the <br> tag. This is an empty or self-closing tag, meaning it doesn’t need a closing tag. It’s useful for writing addresses, poems, or whenever you need a manual line break in the content.

After writing your HTML code with headings and paragraphs in a text editor like Notepad, it’s important to save the file correctly. To do this, click on File > Save As, type a name like headings.html, and make sure to select All Files in the "Save as type" dropdown. Add the .html extension at the end of your file name. Also, choose UTF-8 encoding if it is available. Once saved, you can open the file in any web browser to see how your headings and paragraphs appear.

Lesson 4: Paragraph Formatting

Jump to video lesson

HTML allows you to format specific parts of your paragraph text to highlight important information, add emphasis, or show changes. These formatting tags help improve readability and meaning for both sighted users and screen reader users.

The <strong> tag is used to give strong importance to the text. Most browsers display it in bold, and screen readers also give it extra emphasis. It is preferred over just making text bold because it conveys meaning, not just style.

The <b> tag also makes the text bold, but it is used for styling only without any additional importance or meaning. Use this when you want the text to stand out visually, but not semantically.

The <i> tag is used to italicize text, usually for technical terms, foreign words, or names of books. Like <b>, it is a visual style and doesn't add any meaning to screen readers.

The <em> tag is used to emphasize text. It is usually displayed in italics and screen readers pronounce it with added stress. It is more meaningful than just using <i> for styling.

The <ins> tag is used to indicate inserted text, often shown with an underline. It is useful when showing changes or additions in a paragraph, especially in edited documents.

The <del> tag shows deleted text, usually displayed with a line through it (strikethrough). It can be used to show corrections or revisions in your content.

Another useful formatting tag is <mark>, which highlights text (typically with a yellow background) to draw the reader’s attention to something important.

You can also use <sub> for subscript text (like H2O) and <sup> for superscript text (like x2) within your paragraphs when writing scientific or mathematical content.

These formatting tags make your paragraphs more expressive and accessible. Remember to use semantic tags like <strong> and <em> when the meaning is important, not just the appearance.

Lesson 6: Organizing Your Content with HTML Lists

Jump to video lesson

Lists in HTML are used to group related items together in an organized way. They help structure content for better readability and accessibility. HTML supports three main types of lists: unordered lists, ordered lists, and definition (or description) lists.

An unordered list is used when the sequence of items does not matter. It displays items with bullet points. To create it, you use the <ul> tag, and each item in the list is added using the <li> tag.

An ordered list is used when the order of the items is important, such as in step-by-step instructions, rankings, or procedures. It displays items with numbers or letters by default. It is created using the <ol> tag, and items are listed inside <li> tags.

A definition list (also known as a description list) is used to display terms and their corresponding definitions or descriptions. This type of list is useful for glossaries, FAQs, or any case where you need to present pairs of items. It is created using the <dl> tag (definition list), where each term is placed inside a <dt> tag (definition term), followed by its description in a <dd> tag (definition description).

HTML also allows you to nest lists—placing one list inside another. This is helpful when you need to show sub-items under a main list item, like categories and subcategories.

Using lists properly improves your web page’s structure and enhances accessibility. Screen readers can easily detect list structures, making your content more user-friendly for people with visual impairments.

Lesson 7: Connecting Your Web Pages with HTML Hyperlinks

Jump to video lesson

HTML Hyperlinks allow users to navigate from one page to another or to different sections of the same page. Hyperlinks are created using the <a> (anchor) tag in HTML.

The <a> tag uses the href attribute to specify the destination of the link. When users click the link, they are taken to the page or section specified in the href.

Example:
<a href="https://www.example.com">Visit Example Website</a>

You can also use hyperlinks to link to internal pages of your website. For example:
<a href="contact.html">Contact Us</a>

Target attribute:
Use target="_blank" to open the link in a new tab.
Example:
<a href="https://www.google.com" target="_blank">Open Google in new tab</a>

Link to a specific section in the same page:
You can link to an element with an id using #idname.
Example:
<a href="#about">Go to About Section</a>
and later in the page:
<h2 id="about">About Us</h2>

Types of Hyperlinks:
1. External Link – opens another website.
2. Internal Link – opens a different page of your own website.
3. Anchor Link – jumps to a specific section of the current page.

Hyperlinks are an essential part of web navigation and help connect content across the web.

Lesson 8: Crafting Accessible Tables in HTML

Jump to video lesson

HTML tables are used to organize data into rows and columns, making the information easy to read and compare. Tables are especially useful for displaying structured data like schedules, price lists, or comparison charts.

To create a table in HTML, the <table> element is used. Inside the table, rows are defined using <tr> (table row), and within each row, cells are defined using either <th> (table header) for headings or <td> (table data) for regular cells.

Making tables accessible is very important for screen reader users. To improve table accessibility:

• Use the <caption> tag to provide a summary or title for the table.
• Use <th> to identify column and row headers.
• Add the scope attribute (e.g., scope="col" or scope="row") to <th> elements to specify which part of the table the header applies to.
• Use the summary attribute or visually hidden text (for modern HTML) to describe complex tables.
• Avoid using tables for layout purposes; instead, use CSS for page layout.

Proper use of these semantic elements ensures that assistive technologies can accurately interpret the table’s structure and present it meaningfully to users.

Lesson 9: HTML Forms part 1

Jump to video lesson

HTML forms are used to collect user input on a webpage. They allow users to enter data, such as their name, email, or date of birth, which can then be processed by the server or handled with client-side scripts. A form is created using the <form> element, and it can include various input fields depending on the type of information required.

In this part, we will focus on the most common form input types: text, email, tel, date, password, number, and the submit button.

The text input type allows users to enter plain text, such as their name or address. The email input type ensures that the entered text follows the correct email format, which helps in validating email addresses before submission. The tel input type is used for phone numbers and can display a numeric keypad on mobile devices for easier input. The date input type provides a date picker so that users can choose a date instead of typing it manually.

The password input type hides the characters as the user types, making it suitable for sensitive information like login credentials. The number input type allows only numeric values and may include up and down arrows for adjusting the value. Finally, the submit button is used to send the form data to the specified server or script for processing.

For better accessibility, always associate form fields with descriptive labels using the <label> element. This helps screen readers announce the purpose of each input. You should also use clear placeholder text or instructions, and ensure that error messages are provided if the input is invalid. Making forms accessible ensures that all users, including those using assistive technologies, can interact with your form easily.

Lesson 10: HTML Forms part 2

Jump to video lesson

Radio buttons, checkboxes, and combo boxes are form controls that allow users to select one or more options from a set. Radio buttons are used when only one option can be chosen from a group. They should always share the same name attribute so that only one button can be selected at a time. Each radio button should have a clearly associated label, which can be provided using the <label> element.

Checkboxes are used when the user can select multiple options from a list. Each checkbox should also be paired with a descriptive label, ensuring that assistive technologies can convey the purpose of the control. When there are multiple checkboxes for related options, it is good practice to group them within the <fieldset> element, with a <legend> that describes the group.

Combo boxes, or drop-down lists, are created using the <select> element, with one or more <option> elements inside. They allow users to select from a predefined list of choices. When creating a combo box, provide a meaningful label and avoid using generic text like "Select one" without context. If the list is long, consider grouping related options with the <optgroup> element for better accessibility. Always ensure that combo boxes are keyboard accessible and screen reader friendly.

To make these controls more accessible, always use clear and descriptive labels, ensure keyboard navigation works smoothly, and group related controls logically. Using semantic HTML elements like <label>, <fieldset>, and <legend> helps screen readers convey the form structure accurately to users.

Lesson 11: Enhancing Your Web Pages with Accessible Audio and Video

Jump to video lesson

Multimedia elements such as audio and video allow us to add sound, music, and moving images to a web page. The <audio> element is used to embed audio files, such as music tracks, podcasts, or sound effects. The <video> element is used to display videos directly in the browser without needing extra plugins. Both elements require a src attribute to define the media file and can include controls for play, pause, volume, and seeking.

To make multimedia accessible, always include the controls attribute so users can operate playback with a mouse, keyboard, or touch screen. For audio, provide a transcript of the spoken content so users who cannot hear can still understand the message. For video, provide captions for spoken words and important sounds, and if possible, include audio descriptions for visually impaired users. This ensures that everyone can access the media content regardless of their abilities.

You can also provide multiple file formats, such as MP3, OGG, and WAV for audio, or MP4, WebM, and OGV for video, to ensure that the media works in all browsers. Additionally, make sure that auto-playing media is avoided or muted by default, as it can be disruptive for users, especially those using screen readers. Accessibility guidelines like WCAG recommend giving users full control over media playback.

Lesson 12: Meta tags

Jump to video lesson

In HTML, the <meta> tag is placed inside the <head> section and is used to provide metadata about the web page. Metadata means “data about data,” which is information that helps browsers, search engines, and other tools understand more about your page. Meta tags do not appear visually on the page but are important for accessibility, SEO (Search Engine Optimization), and proper browser rendering.

One of the most important meta tags is the <meta charset="UTF-8"> tag. This tag tells the browser which character encoding to use. UTF-8 is the most widely used encoding, and it supports almost all characters from all languages, making it suitable for multilingual websites.

Another common meta tag is the viewport meta tag: <meta name="viewport" content="width=device-width, initial-scale=1.0">. This tag is essential for responsive web design. It instructs the browser to adjust the page’s width and scaling according to the device’s screen size, so your page looks good on both mobile and desktop devices.

There are also meta tags for descriptions and keywords. For example: <meta name="description" content="A brief description of the web page"> helps search engines display a meaningful summary in search results. Similarly, <meta name="keywords" content="HTML, tutorial, accessibility"> provides search keywords, though modern search engines rely less on this tag.

From an accessibility perspective, the charset meta tag ensures that text displays correctly for assistive technologies such as screen readers. The viewport meta tag ensures that zoom and scaling work properly for users with low vision. Additionally, using clear and relevant descriptions in meta tags helps screen readers and search engines present better information about the page to the user.
